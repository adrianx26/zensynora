# Agent Swarm Guide

## Overview

Agent Swarms is a powerful multi-agent coordination system for MyClaw that enables multiple AI agents to collaborate on complex tasks. Instead of relying on a single agent, swarms distribute work across specialized agents using various coordination strategies.

## 🎯 Swarm Coordinator

### What is the Swarm Coordinator?

The **Swarm Coordinator** is a special agent role in the **Hierarchical Strategy** that:
1. **Plans** - Analyzes the task and creates an execution plan
2. **Delegates** - Assigns subtasks to worker agents
3. **Monitors** - Tracks worker progress
4. **Synthesizes** - Combines worker outputs into a coherent final result

### When to Use a Coordinator?

Use a coordinator when:
- Tasks need to be broken into subtasks
- Different agents should handle different aspects
- Results need intelligent combination
- You need a "project manager" for complex work

### Example: Hierarchical Code Review

```
User assigns task: "Review this authentication module"

Coordinator (lead_dev) actions:
1. Creates plan:
   - security_expert → Check SQL injection
   - performance_reviewer → Check efficiency
   - style_checker → Check code style

2. Workers execute in parallel

3. Coordinator synthesizes:
   "Security: 2 issues found...
    Performance: Acceptable...
    Style: Minor formatting needed..."
```

### Configuration with Coordinator

```
swarm_create(
    name="code_review_team",
    strategy="hierarchical",
    coordinator="lead_developer",    ← Coordinator agent
    workers="security_expert,performance_reviewer,style_checker",
    aggregation="synthesis"
)
```

## Configuration

Add to your `~/.myclaw/config.json`:

```json
{
  "swarm": {
    "enabled": true,
    "max_concurrent_swarms": 3,
    "default_strategy": "parallel",
    "default_aggregation": "synthesis",
    "timeout_seconds": 300,
    "swarm_memory_limit": 50
  }
}
```

### Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| `enabled` | `true` | Enable/disable swarm functionality |
| `max_concurrent_swarms` | `3` | Maximum parallel swarms per user |
| `default_strategy` | `"parallel"` | Default execution strategy |
| `default_aggregation` | `"synthesis"` | Default result aggregation |
| `timeout_seconds` | `300` | Maximum execution time |
| `swarm_memory_limit` | `50` | Max messages stored per swarm |

## Best Practices

### 1. Agent Selection

**Complementary Skills:**
- Research team: `researcher1, researcher2, researcher3` (different specialties)
- Code review: `security_expert, performance_reviewer, style_checker`
- Content: `researcher, writer, editor`

**Coordinator Selection:**
- Should have good planning capabilities
- Needs to understand the big picture
- Prefer models with strong reasoning

### 2. Strategy Selection Guide

| Task Type | Strategy | Why |
|-----------|----------|-----|
| Research/Broad Analysis | **Parallel** | Multiple perspectives simultaneously |
| Content Creation | **Sequential** | Progressive refinement |
| Complex Multi-part Tasks | **Hierarchical** | Orchestrated subtasks |
| Decision Making | **Voting** | Consensus validation |
| Code Review | **Parallel** or **Hierarchical** | Different aspects or orchestrated |
| Data Processing | **Sequential** | Step-by-step transformation |

### 3. Aggregation Selection

| Goal | Method | Example |
|------|--------|---------|
| Get diverse perspectives | **Concatenation** | Research reports |
| Find best single answer | **Best Pick** | Code solutions |
| Quick agreement | **Consensus** | Yes/No decisions |
| Comprehensive summary | **Synthesis** | Final reports |

### 4. Resource Management

```python
# Check current usage
swarm_stats()

# Clean up old swarms
swarm_list(status="completed")  # Find completed
swarm_terminate("swarm_old123")  # Clean up
```

## API Reference

### Core Functions

#### `swarm_create()`
Create a new swarm.

**Parameters:**
- `name` (str): Descriptive name for the swarm
- `strategy` (str): One of: `parallel`, `sequential`, `hierarchical`, `voting`
- `workers` (str): Comma-separated list of agent names
- `coordinator` (str, optional): Coordinator agent name (required for hierarchical)
- `aggregation` (str): One of: `consensus`, `best_pick`, `concatenation`, `synthesis`
- `user_id` (str, optional): User identifier

**Returns:** Swarm ID (e.g., `swarm_abc123def456`)

---

#### `swarm_assign()`
Execute a task with the swarm.

**Parameters:**
- `swarm_id` (str): Swarm identifier
- `task` (str): Task description/prompt
- `user_id` (str, optional): User identifier

**Returns:** Formatted results including:
- Individual agent outputs
- Confidence score
- Execution time
- Aggregated final result

---

#### `swarm_status()`
Get current swarm status.

**Parameters:**
- `swarm_id` (str): Swarm identifier

**Returns:** Status information (pending, running, completed, failed, terminated)

---

#### `swarm_result()`
Get detailed execution result.

**Parameters:**
- `swarm_id` (str): Swarm identifier

**Returns:** Complete result with individual agent outputs

---

#### `swarm_terminate()`
Force stop a running swarm.

**Parameters:**
- `swarm_id` (str): Swarm identifier

---

#### `swarm_list()`
List all swarms.

**Parameters:**
- `status` (str, optional): Filter by status
- `user_id` (str, optional): User identifier

---

#### `swarm_stats()`
Get usage statistics.

**Parameters:**
- `user_id` (str, optional): User identifier

**Returns:** Statistics including counts, avg confidence, execution times

## Troubleshooting

### Common Issues

#### "Maximum concurrent swarms reached"

**Cause:** Hit the `max_concurrent_swarms` limit

**Solutions:**
1. Wait for existing swarms to complete
2. Terminate unused swarms:
   ```python
   swarm_list(status="running")
   swarm_terminate("swarm_xxx")
   ```
3. Increase limit in config:
   ```json
   {"swarm": {"max_concurrent_swarms": 5}}
   ```

#### "Agent not found in registry"

**Cause:** Agent name doesn't exist in the agent registry

**Solution:** Check available agents and correct the name:
```python
# List available agents
# (Check your config.json agents.named section)
```

#### Swarm execution timeout

**Cause:** Task took longer than `timeout_seconds`

**Solutions:**
- Break task into smaller parts
- Increase timeout in config
- Use fewer workers for faster execution

#### Low confidence scores

**Cause:** Agents disagree significantly or quality varies

**Solutions:**
- Try different aggregation method
- Provide clearer, more specific task description
- Add more workers (for voting strategy)
- Use coordinator for better synthesis

## Storage

Swarm data is stored in `~/.myclaw/swarm.db`:

### Tables

**swarms**
- Swarm configurations and status
- Strategy, workers, coordinator settings

**swarm_tasks**
- Individual task assignments
- Input/output data

**swarm_messages**
- Inter-agent communication log
- Useful for debugging

**swarm_results**
- Execution results
- Individual and aggregated outputs

### Cleanup

Old swarms are automatically cleaned up after 30 days. You can manually clean up:

```python
# In Python code
from myclaw.swarm import SwarmStorage
storage = SwarmStorage()
deleted = storage.cleanup_old_swarms(days=7)
print(f"Cleaned up {deleted} old swarms")
```

## Advanced Usage

### Custom Strategies

You can extend the system with custom strategies by inheriting from `BaseStrategy`:

```python
from myclaw.swarm.strategies import BaseStrategy

class MyCustomStrategy(BaseStrategy):
    async def execute(self, task, workers, coordinator=None):
        # Your custom logic here
        pass
```

### Direct Orchestrator Access

For programmatic control:

```python
from myclaw.swarm import SwarmOrchestrator, SwarmConfig, SwarmStrategy

orchestrator = SwarmOrchestrator(agent_registry, config)

# Create swarm
config = SwarmConfig(
    name="my_swarm",
    strategy=SwarmStrategy.PARALLEL,
    workers=["agent1", "agent2"]
)
swarm_id = await orchestrator.create_swarm(config)

# Execute and get result object directly
result = await orchestrator.execute_task(swarm_id, "My task")
print(result.confidence_score)
print(result.individual_results)
```

## Key Concepts

### What is an Agent Swarm?

An Agent Swarm is a group of AI agents working together under a specific coordination strategy to solve problems that are too complex or multifaceted for a single agent.

### Swarm Strategies

1. **Parallel** - All agents work simultaneously on the same task
   - Best for: Multi-perspective analysis, brainstorming, independent research
   - Example: Three researchers independently investigating the same topic

2. **Sequential** - Agents work in a pipeline, output feeds to next input
   - Best for: Multi-stage processing, content creation workflows
   - Example: Outline → Draft → Edit → Review

3. **Hierarchical** - Coordinator agent delegates subtasks to workers
   - Best for: Complex multi-part tasks requiring orchestration
   - Example: Project manager assigning tasks to team members

4. **Voting** - Multiple agents solve the same problem, consensus wins
   - Best for: Decision making, quality validation, uncertain problems
   - Example: Multiple agents rating code quality

### Aggregation Methods

After agents complete their work, results are combined using:

- **Consensus** - Most common answer wins
- **Best Pick** - Highest quality response selected
- **Concatenation** - All outputs combined with separators
- **Synthesis** - Intelligent combination (default)

## Using Agent Swarms

### Creating a Swarm

```
swarm_create(
    name="research_team",
    strategy="parallel",
    workers="researcher1,researcher2,researcher3",
    aggregation="synthesis"
)
```

Response:
```
✅ Swarm created successfully!
   ID: swarm_abc123def456
   Name: research_team
   Strategy: parallel
   Workers: researcher1, researcher2, researcher3
   Coordinator: N/A
   Aggregation: synthesis
```

### Assigning Tasks

```
swarm_assign(
    swarm_id="swarm_abc123def456",
    task="Research the latest developments in quantum computing in 2024"
)
```

Response:
```
🐝 Swarm Execution Complete
   Swarm: swarm_abc123def456
   Aggregation: synthesis
   Confidence: 0.85
   Execution Time: 12.34s

📊 Individual Results:
   ✅ researcher1: 1,234 chars
   ✅ researcher2: 987 chars
   ✅ researcher3: 1,567 chars

🎯 Final Result:
[Combined and synthesized output from all agents]
```

### Managing Swarms

Check status:
```
swarm_status(swarm_id="swarm_abc123def456")
```

Get result:
```
swarm_result(swarm_id="swarm_abc123def456")
```

List all swarms:
```
swarm_list()
swarm_list(status="running")
```

Get statistics:
```
swarm_stats()
```

Terminate a running swarm:
```
swarm_terminate(swarm_id="swarm_abc123def456")
```

## Configuration

Add to your `~/.myclaw/config.json`:

```json
{
  "swarm": {
    "enabled": true,
    "max_concurrent_swarms": 3,
    "default_strategy": "parallel",
    "default_aggregation": "synthesis",
    "timeout_seconds": 300
  }
}
```

### Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| `enabled` | `true` | Enable/disable swarm functionality |
| `max_concurrent_swarms` | `3` | Maximum parallel swarms per user |
| `default_strategy` | `"parallel"` | Default execution strategy |
| `default_aggregation` | `"synthesis"` | Default result aggregation |
| `timeout_seconds` | `300` | Maximum execution time |

## Use Cases

### 1. Research Team (Parallel)

Create multiple agents with different specialties to research a topic:

```
swarm_create(
    name="ai_research_team",
    strategy="parallel",
    workers="tech_analyst,market_researcher,ethics_reviewer",
    aggregation="synthesis"
)

swarm_assign(
    swarm_id="swarm_xxx",
    task="Analyze the impact of AI on healthcare in 2024"
)
```

Each agent provides their perspective, results are synthesized.

### 2. Content Pipeline (Sequential)

Create a content creation workflow:

```
swarm_create(
    name="blog_pipeline",
    strategy="sequential",
    workers="outline_writer,content_writer,editor,seo_optimizer",
    aggregation="concatenation"
)

swarm_assign(
    swarm_id="swarm_xxx",
    task="Write a blog post about Python best practices"
)
```

Output flows: Outline → Content → Edited Content → SEO-Optimized

### 3. Code Review (Hierarchical)

Multi-aspect code review with a lead:

```
swarm_create(
    name="code_review",
    strategy="hierarchical",
    coordinator="lead_developer",
    workers="security_expert,performance_reviewer,style_checker",
    aggregation="synthesis"
)

swarm_assign(
    swarm_id="swarm_xxx",
    task="Review this authentication module for issues"
)
```

Lead coordinates reviews, synthesizes findings.

### 4. Decision Voting (Voting)

Get consensus on decisions:

```
swarm_create(
    name="architecture_vote",
    strategy="voting",
    workers="architect1,architect2,architect3,architect4",
    aggregation="consensus"
)

swarm_assign(
    swarm_id="swarm_xxx",
    task="Should we use microservices or monolith for this project?"
)
```

Consensus determines the winning approach.

## Best Practices

### 1. Agent Selection

- Choose agents with complementary skills
- Ensure all agents in a swarm use appropriate models
- Consider agent specialties when assigning tasks

### 2. Strategy Selection

| Task Type | Recommended Strategy |
|-----------|---------------------|
| Research/Broad Analysis | Parallel |
| Content Creation | Sequential |
| Complex Multi-part Tasks | Hierarchical |
| Decision Making | Voting |
| Code Review | Hierarchical or Parallel |

### 3. Aggregation Selection

| Goal | Recommended Aggregation |
|------|------------------------|
| Get diverse perspectives | Concatenation |
| Find best answer | Best Pick or Consensus |
| Comprehensive report | Synthesis |
| Quick decision | Consensus |

### 4. Resource Management

- Monitor `swarm_stats()` to track usage
- Terminate unused swarms with `swarm_terminate()`
- Stay within `max_concurrent_swarms` limit

## Troubleshooting

### "Maximum concurrent swarms reached"

Either:
- Wait for existing swarms to complete
- Terminate unused swarms: `swarm_terminate(swarm_id)`
- Increase limit in config: `max_concurrent_swarms`

### "Agent not found in registry"

Ensure the agent names in `workers` parameter match configured named agents in your config.

### Swarm execution timeout

- Reduce task complexity
- Increase `timeout_seconds` in config
- Use fewer workers

### Low confidence scores

- Try different aggregation method
- Ensure agents have clear, specific prompts
- Add more workers for voting strategy

## API Reference

### swarm_create

Create a new swarm.

**Parameters:**
- `name` (str): Descriptive name
- `strategy` (str): `parallel`, `sequential`, `hierarchical`, `voting`
- `workers` (str): Comma-separated agent names
- `coordinator` (str, optional): Coordinator agent (required for hierarchical)
- `aggregation` (str): `consensus`, `best_pick`, `concatenation`, `synthesis`
- `user_id` (str, optional): User identifier

### swarm_assign

Execute task with swarm.

**Parameters:**
- `swarm_id` (str): Swarm identifier
- `task` (str): Task description
- `user_id` (str, optional): User identifier

### swarm_status

Get swarm status.

**Parameters:**
- `swarm_id` (str): Swarm identifier

### swarm_result

Get execution result.

**Parameters:**
- `swarm_id` (str): Swarm identifier

### swarm_terminate

Stop a running swarm.

**Parameters:**
- `swarm_id` (str): Swarm identifier

### swarm_list

List swarms.

**Parameters:**
- `status` (str, optional): Filter by status
- `user_id` (str, optional): User identifier

### swarm_stats

Get usage statistics.

**Parameters:**
- `user_id` (str, optional): User identifier

## Storage

Swarm data is stored in `~/.myclaw/swarm.db`:

- **swarms** - Swarm configurations and status
- **swarm_tasks** - Individual task assignments
- **swarm_messages** - Inter-agent communication
- **swarm_results** - Execution results

Old swarms are automatically cleaned up after 30 days.
