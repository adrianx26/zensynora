ZenSynora Architecture

Multi-tenant agent platform — channels, auth, core engine, providers, storage, tools, marketplace, observability

Channels & Auth

  • 5 channel adapters: Telegram, WhatsApp, Discord, CLI, Web UI
  • JWTAuthenticator + OAuth2Callback
  • UserContext via contextvars (per-request tenant scoping)

Core Engine

  • Agent orchestrates Router, Context, Executor, Response
  • Async tool dispatch with streaming output
  • Per-tenant memory and knowledge isolation

Resilience & LLM

  • CircuitBreaker per provider
  • FallbackChain across Ollama / OpenAI / Anthropic / Gemini / Groq
  • SemanticCache for repeated-query dedup

Storage

  • Memory: per-tenant conversation log
  • KnowledgeDB: FTS5 + dedicated executor
  • Vector store: memory / sqlite / qdrant
  • BaseTTLCache for in-process caching

Tools & Marketplace

  • Tool Registry, browser_*, register_tool sandbox
  • PromptRegistry, repair_json
  • Marketplace sources: OpenClaw, GitHub, HTTP, Local
  • MarketplaceClient with HMAC signature verify

Observability

  • OpenTelemetry spans (graceful no-op if missing)
  • PII scrubber on log/trace pipelines
  • CostTracker — per-call provider accounting
