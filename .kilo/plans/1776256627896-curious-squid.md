## Phase 1.2: Auto Health Check on Startup

### Objective and Success Criteria

**Objective:** Wire up the `scan_on_startup` configuration option to trigger MedicAgent health checks during gateway startup.

**Success Criteria (from gap report):**
- `scan_on_startup` config field triggers actual health check on startup
- `gateway.start()` instantiates `MedicAgent` when configured
- Health check results are logged appropriately
- No startup failures if health check fails (graceful degradation)

### Step-by-Step Strategy

1. **Analyze current state:** `scan_on_startup` exists in config but `gateway.py` never instantiates `MedicAgent`
2. **Design integration point:** Add MedicAgent initialization and conditional scan in `gateway.start()`
3. **Implement startup hook:** Create `_run_startup_health_check()` function
4. **Add config validation:** Ensure `medic.enabled` is checked before running
5. **Test startup scenarios:** Verify behavior with/without config enabled
